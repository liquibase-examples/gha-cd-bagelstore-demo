Automated PR Validation

GitHub Actions Runs Policy Checks:

üõ°Ô∏è 12 BLOCKER-Level Checks
‚úì Prevent DROP TABLE/COLUMN
‚úì Prevent TRUNCATE operations
‚úì Require rollback capability
‚úì Detect risky GRANT/REVOKE
‚úì Enforce table column limits
‚úì Check for proper indexes
‚úÖ All checks passed ‚Üí PR can be merged
‚ùå BLOCKER violation found ‚Üí PR blocked until fixed
Policy checks run before merge, blocking risky changes. All checks set to BLOCKER severity. Reports uploaded to S3 and GitHub Actions for review.

Main Branch CI: Parallel Build

Database Workflow

  1. Validate changelog
  2. Run policy checks
  3. Create changelog.tar.gz
  4. Upload to GitHub Packages
  5. Upload reports to S3

Application Workflow

  1. Extract version from git tag
  2. Build Docker image
  3. Tag with version
  4. Push to AWS Public ECR
  5. Tag as 'latest'

‚Üì

Both jobs complete ‚Üí Trigger Harness webhook

Parallel builds for speed. Database and application artifacts created simultaneously. Both artifacts tagged with same version for coordinated deployment.

Governance with Liquibase Policy Checks

12 Checks Configured at BLOCKER Severity

Exit code 4 = Pipeline fails, PR cannot merge

Destructive Operations
  • ChangeDropTableWarn
  • ChangeDropColumnWarn
  • ChangeTruncateTableWarn
Safety Requirements
  • RollbackRequired
  • ModifyDataTypeWarn
  • CheckTablesForIndex
Security & Permissions
  • SqlGrantWarn
  • SqlGrantOptionWarn
  • SqlGrantAdminWarn
  • SqlRevokeWarn
Best Practices
  • SqlSelectStarWarn
  • TableColumnLimit (50 max)
Policy checks enforce database governance automatically. Prevent common mistakes and risky changes. Configuration stored in S3, referenced by all workflows.
